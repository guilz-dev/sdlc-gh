# Support

## Where to ask what

- Usage questions: use the `Usage question` issue template and describe your repository type, selected stack, and where adoption is blocked
- Bug reports: use the `Bug report` issue template with reproduction steps, expected behavior, and actual behavior
- Security reports: follow `SECURITY.md` and do not post exploit details publicly
- Design proposals: use the `Feature request` issue template before large workflow, policy, or architecture changes

## What to include

For adoption or troubleshooting questions, include:

- whether you are using the template directly or bootstrapping into an existing repository
- selected stack: `ts`, `python`, `go`, `ruby`, or `php`
- Node.js version
- which local or GitHub Actions check failed
- relevant logs or screenshots, with secrets removed

## Support boundaries

This repository is a template and reference harness. Maintainers may not be able to provide bespoke support for:

- organization-specific GitHub governance policy
- private Copilot or GitHub Models enablement issues
- custom downstream workflow integrations unrelated to this template
